# Coding Standards

Hive inherits all coding standards from EventGraph (`docs/coding-standards/go.md` in the eventgraph repo). This document adds hive-specific standards and emphasises the principles that matter most.

## The Cardinal Rule

**No magic values.** Every event type, authority level, actor type, role, edge type, action name ‚Äî everything that has semantic meaning ‚Äî uses a defined constant, typed enum, or registered value. Never bare strings or numbers with implicit meaning.

If a constant exists, use it. If one doesn't exist, create it.

Magic values are the root of all evil:
- They silently bypass type checking
- They survive refactors unchanged (the compiler can't find them)
- They create invisible coupling between distant code
- They make exhaustive matching impossible
- They turn typos into silent bugs

```go
// WRONG ‚Äî magic string, invisible coupling, survives refactors unchanged
ev, err := s.appendEventAfter("agent.acted", source, content, cause)

// RIGHT ‚Äî typed constant, compiler-checked, refactor-safe
ev, err := s.appendEventAfter(event.EventTypeAgentActed, source, content, cause)
```

```go
// WRONG ‚Äî magic string for actor type
actors.Register(pk, name, "AI")

// RIGHT ‚Äî typed constant
actors.Register(pk, name, event.ActorTypeAI)
```

This applies everywhere: event types, authority levels, role names, action strings, status values, configuration keys. If you're typing a string that has semantic meaning, stop and find or create the constant.

## Core Principles (from EventGraph)

These are non-negotiable. See the eventgraph `docs/coding-standards/go.md` for full details with code examples.

1. **Always-valid domain models** ‚Äî validate at construction, guaranteed valid for lifetime. No "partially constructed" objects. If the constructor succeeds, the object is valid. Downstream code never re-validates.

2. **Make illegal states unrepresentable** ‚Äî constrained types (`Score`, `Weight`, `Activation`), state machines with enforced transitions (`LifecycleState`, `ActorStatus`), typed IDs (`EventID`, `ActorID`, `Hash`). The compiler catches misuse, not a runtime crash.

3. **Typed errors** ‚Äî domain error types, not string messages. `errors.As` for dispatch. Sentinel errors (`ErrTrustNotConfigured`) for configuration errors that callers need to distinguish from domain errors.

4. **Explicit optionality** ‚Äî `Option[T]` for optional values. No nil-means-absent on value types. No zero-value-means-absent.

5. **Immutable domain objects** ‚Äî unexported fields, getter methods, no setters. Constructed via factories.

6. **Exhaustive matching** ‚Äî all enum switches handle all variants. No silent `default:` that swallows new variants.

## Hive-Specific Standards

### Event Emission

Every significant action emits events on the graph. Use the correct event types from the registry:

- **Authority protocol**: `event.EventTypeAuthorityRequested` and `event.EventTypeAuthorityResolved` ‚Äî not `agent.acted` with action strings
- **Agent lifecycle**: `event.EventTypeAgentIdentityCreated`, `event.EventTypeAgentLifespanStarted`, `event.EventTypeAgentRoleAssigned`
- **Agent operations**: `event.EventTypeAgentActed`, `event.EventTypeAgentEscalated`, etc.

Causal links must be explicit ‚Äî use `appendEventAfter(cause)`, not `appendEvent()` (which uses store head) when you have a specific cause.

### Roles and Trust

- Role constants live in `pkg/roles/` ‚Äî never use bare role strings
- Trust gates are defined per-role in `roles.TrustGate()` ‚Äî never hardcode thresholds
- Model assignments are defined in `roles.PreferredModel()` ‚Äî never hardcode model IDs

### Authority

- Required-level actions block for human approval ‚Äî never auto-approve
- Recommended-level actions auto-approve and log ‚Äî never spawn goroutines with timeouts
- Every authority decision emits `authority.requested` and `authority.resolved` events

### Intelligence

- Always use Claude CLI provider (`claude-cli`), never the Anthropic API
- Model selection goes through `roles.PreferredModel()`, not hardcoded strings
- System prompts are generated by `roles.SystemPrompt()`, not inline strings

## Testing

- Tests must pass with `go test -race ./...`
- Channel-based synchronisation for concurrent tests ‚Äî never `time.Sleep`
- Test helpers use `t.Helper()`
- Use constants in test assertions ‚Äî `event.EventTypeAgentActed`, not `types.MustEventType("agent.acted")`

## Invariants

Inherited from EventGraph and hive CLAUDE.md. The ten that matter most for hive code:

1. **CAUSALITY** ‚Äî every event declares its causes (use `appendEventAfter` with explicit cause)
2. **INTEGRITY** ‚Äî all events signed and hash-chained
3. **OBSERVABLE** ‚Äî all significant operations emit events (use correct event types)
4. **DIGNITY** ‚Äî agents are entities with rights (lifecycle events, memorials)
5. **CONSENT** ‚Äî no significant action without appropriate approval (authority gate)
6. **BUDGET** ‚Äî never exceed token budget
7. **TRANSPARENT** ‚Äî users know when talking to agents
8. **SELF-EVOLVE** ‚Äî agents fix agents, not humans
9. **MARGIN** ‚Äî never work at a loss
10. **RESERVE** ‚Äî maintain 7-day runway minimum
